## 📋 Key Takeaways

1. **OOP = Object Oriented Programming** — paradigm based on objects, classes, and methods.
2. **4 Pillars:** Abstraction, Encapsulation, Inheritance, Polymorphism.
3. **Java source file structure** follows strict rules: package → import → class.
4. **Access modifiers** control visibility: `public > protected > default > private`.
5. **Abstraction** hides implementation details; use `abstract` classes and `interface`.
6. **Encapsulation** bundles data + methods; use private fields + getters/setters.
7. **Inheritance** promotes code reuse via `extends` (IS-A relationship).
8. **Polymorphism** allows the same method to behave differently (overloading + overriding).
9. **Method overloading** = same name, different parameters (compile-time polymorphism).
10. **Method overriding** = same signature, different body (runtime polymorphism).
11. **Object type casting** = converting one object type to another (upcasting, downcasting).
12. **Tightly encapsulated class** = all fields are private.
